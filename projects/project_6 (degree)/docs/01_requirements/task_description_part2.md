## 1. Разработка прототипа

### Исходные требования

- прототип должен покрывать все функции MVP, определённые в первой части проекта;
- используйте элементы предоставленной дизайн-системы.

### Подготовка к работе в Figma

1. Авторизуйтесь в Figma (аккаунт создан на спринте «Проектирование интерфейсов»).
2. Перейдите по ссылке на дизайн-систему.
3. Создайте копию файла: `Duplicate to your drafts` → `Open`.
4. Добавьте страницу «Прототип приложения Stets Home» — это рабочее пространство.
5. Скопируйте нужные компоненты дизайн-системы в рабочий холст.

### Создание прототипа

- соберите сториборды/вайрфреймы для каждой функции;
- используйте фрейм `iPhone 13 mini (375 × 812)` для всех экранов;
- соедините экраны стрелками, показывая переходы пользователя;
- при необходимости снабдите переходы короткими пояснениями.

**Рекомендация:** изучите конкурентов (Google Home, Apple Home), чтобы вдохновиться UX-решениями.

### Задание со звёздочкой (по желанию)

- сделайте прототип интерактивным, используя инструменты прототипирования Figma;
- для тренировки используйте отдельный файл с инструкциями — не забудьте создать его копию.

### После завершения прототипа

1. Нажмите `Share` → `Can view` → `Copy link`.
2. Сохраните ссылку в текстовый файл в общей папке диплома.
3. Переходите к разработке ПМИ.

## 2. Программа и методика испытаний (ПМИ)

1. Создайте документ «ПМИ Stets Home».
2. Вставьте таблицу по шаблону курса, включив:
   - объект испытаний;
   - цель испытаний;
   - требования к программе;
   - требования к документации;
   - средства и порядок испытаний;
   - методику испытаний.
3. Заполните все пункты, опираясь на разработанный прототип и выявленные функции.
4. После завершения ПМИ переходите к записи видео.

## 3. Видеозащита

### Перед записью убедитесь, что готовы

- интерактивный или статический прототип;
- заполненная ПМИ.

### Структура выступления (20–25 минут)

- 2–3 минуты — вступление;
- ~15 минут — основная часть: демонстрация прототипа по шагам ПМИ, аргументация решений, показ диаграмм и словаря данных (при необходимости);
- 2–3 минуты — выводы и итоги.

### Рекомендации по записи в Zoom

- записывайте с компьютера или ноутбука, проверьте камеру и звук;
- запустите запись (`Record`);
- для демонстрации прототипа включите `Share Screen` → выберите экран → `Поделиться`;
- по окончании остановите демонстрацию (`Stop Share`) и запись (`Stop Recording`);
- после завершения встречи дождитесь автоматического сохранения видео, переименуйте файл в формате `Имя_Фамилия_НомерКогорты.mp4`.

### Что сделать после записи

1. Загрузите видео в папку первой части диплома.
2. Создайте текстовый файл со ссылкой на прототип Figma.
3. Добавьте документ ПМИ.
4. Проверьте наличие всех артефактов первой части — ревьюеру нужен полный комплект.
5. Убедитесь, что доступ к папке открыт, затем нажмите «Скопировать ссылку» и отправьте на проверку.

### Дополнительные материалы

- изучите пример видеозащиты из методических материалов курса — он показывает ожидаемый формат выступления;
- держите под рукой диаграммы, словарь данных и карту пользовательских историй, чтобы быстро отвечать на вопросы заказчика.

Что входит в дизайн-систему
Самой простой дизайн-системой можно назвать набор UI-kit + цвета + шрифты.
UI-kit — это набор стандартных элементов экрана, которые часто встречаются в приложении. Из них, как из конструктора, можно собирать новые экраны.
В файле с дизайн-системой вы найдёте заготовки элементов, цвета и ссылки на шрифты, которые необходимо использовать в прототипе.

Преимущество использования дизайн-системы заключается в использовании мастер-компонентов. При изменении мастер-компонента в дизайн-системе изменяются все его копии. При изменении стандартного цвета (стиля) меняется цвет всех объектов, где он применён.
