# План выступления для видеозащиты дипломного проекта

**Проект:** Stets Home - система управления умным домом  
**Длительность:** 20-25 минут  
**Формат:** Видеозапись в Zoom с демонстрацией прототипа

---

## Структура выступления

### 1. Вступление (2-3 минуты)

#### Приветствие и представление
- Поприветствовать заказчика и ревьюера
- Представиться: "Меня зовут Антон Курганский"
- Назвать проект: "Сегодня я представляю дипломный проект - систему управления умным домом Stets Home"

#### Краткое описание проекта
- **Что это:** Мобильное приложение для управления умным домом
- **Цель проекта:** Создать удобное и интуитивное приложение для управления устройствами умного дома
- **Целевая аудитория:** Два типа пользователей - новички (Михаил) и эксперты (Алёна)

#### Описание процесса работы
- "В первой части дипломного проекта я провел анализ требований, создал карту пользовательских историй, разработал диаграммы потоков данных и ER-модель"
- "Во второй части я создал прототип приложения в Figma, разработал программу и методику испытаний"
- "Сегодня я продемонстрирую прототип и покажу, как реализованы все функции MVP"

#### Обзор структуры презентации
- "Презентация состоит из трех частей:"
  1. "Краткое вступление и обзор проекта" 
  2. "Демонстрация прототипа по шагам интегрированного ПМИ - покажу все функции MVP и дополнительные функции"
  3. "Итоги и выводы"

**Ключевые моменты:**
- Уверенность и четкость речи
- Показать структурированность работы
- Создать позитивный настрой

---

### 2. Основная часть (15 минут)

#### Демонстрация прототипа по шагам ПМИ

**Структура демонстрации:** Последовательное прохождение по всем трем активностям

#### АКТИВНОСТЬ 1: Войти в приложение (3-4 минуты)

**US-001: Регистрация нового пользователя**
- Открыть экран входа
- Показать переход на экран регистрации
- Продемонстрировать заполнение формы (имя, email, пароль)
- Показать валидацию полей в реальном времени:
  - Невалидный email
  - Пароли не совпадают
  - Невалидный пароль
- Продемонстрировать успешную регистрацию
- **Аргументация:** "Регистрация включает валидацию email и пароля на клиенте для улучшения UX. Пароль должен содержать 8-16 символов с обязательными строчными и прописными буквами для безопасности. Проверка совпадения паролей предотвращает ошибки ввода"

**US-003: Вход в приложение**
- Вернуться на экран входа
- Продемонстрировать успешный вход с валидными данными
- Показать обработку ошибок:
  - Неверный пароль
  - Несуществующий email
- **Аргументация:** "Обработка ошибок входа важна для безопасности. Раздельные сообщения для неверного пароля и несуществующего email помогают пользователю понять проблему"

**US-004: Восстановление пароля**
- Показать экран восстановления пароля
- Продемонстрировать запрос восстановления (отправка 4-значного кода)
- Показать экран ввода кода и нового пароля
- Продемонстрировать обработку ошибок:
  - Просроченный код (старше 5 минут)
  - Неверный код
- Показать возможность повторной отправки кода
- **Аргументация:** "Восстановление пароля через 4-значный код - быстрый и безопасный подход. Код действует 5 минут, что обеспечивает баланс между безопасностью и удобством. Возможность повторной отправки улучшает UX"

**US-005: Изменение email**
- Перейти в настройки → "Личные данные"
- Показать экран изменения email
- Продемонстрировать процесс:
  - Ввод нового email
  - Отправка кода подтверждения
  - Ввод кода и подтверждение
- **Аргументация:** "Изменение email через код подтверждения - безопасный подход. Код отправляется сразу после ввода нового email, что ускоряет процесс по сравнению с письмом"

**US-006: Выход из приложения**
- Показать кнопку выхода в профиле
- Продемонстрировать выход и возврат на экран входа
- **Аргументация:** "При выходе очищаются локальные кеши и токены, сессия закрывается на сервере"

#### АКТИВНОСТЬ 2: Организовать умный дом (5-6 минут)

**US-007: Создание комнат**
- Показать главный экран с кнопкой "Добавить комнату"
- Продемонстрировать создание комнаты (название, тип, иконка)
- Показать валидацию (пустое название, превышение лимита 10 комнат)
- **Аргументация:** "Комнаты помогают структурировать управление устройствами. Лимит в 10 комнат достаточен для большинства домов"

**US-008: Редактирование комнат**
- Показать экран редактирования комнаты
- Продемонстрировать изменение названия, типа, иконки
- **Аргументация:** "Редактирование комнат позволяет пользователям адаптировать структуру под свои нужды"

**US-009: Удаление комнат**
- Показать попытку удаления пустой комнаты (успешно)
- Показать попытку удаления комнаты с устройствами (предупреждение)
- **Аргументация:** "Защита от удаления комнат с устройствами предотвращает потерю данных и улучшает UX"

**US-010: Добавление устройства через QR-код**
- Показать вкладку "Устройства"
- Продемонстрировать выбор "Добавить по QR-коду"
- Показать экран сканера QR-кода
- Показать экран подтверждения данных устройства
- Продемонстрировать обработку ошибок:
  - QR другого производителя
  - Превышение лимита (100 устройств)
- **Аргументация:** "QR-код - самый быстрый способ добавления устройств. Это критично для достижения метрики успеха: время онбординга < 5 минут. Проверка производителя QR-кода предотвращает добавление несовместимых устройств"

**US-011: Добавление устройства вручную**
- Показать альтернативный способ - ввод кода вручную
- Продемонстрировать процесс:
  - Ввод 12-значного ID устройства
  - Выбор типа устройства из справочника
  - Указание пользовательского названия (опционально)
- **Аргументация:** "Ручной ввод необходим, если QR-код поврежден. Выбор типа из справочника обеспечивает корректную категоризацию устройств. Валидация формата предотвращает ошибки"

**US-012: Предотвращение дубликатов устройств**
- Показать попытку добавить уже существующее устройство
- Продемонстрировать сообщение об ошибке
- **Аргументация:** "Проверка дубликатов на уровне дома обеспечивает целостность данных"

**US-013: Привязка устройства к комнате**
- Показать экран настройки устройства
- Продемонстрировать выбор комнаты из списка
- Показать опцию "Без комнаты"
- **Аргументация:** "Гибкость привязки устройств позволяет пользователям организовать дом по своему усмотрению"

**US-014: Удаление устройства**
- Показать карточку устройства
- Продемонстрировать удаление через свайп влево
- Показать подтверждение удаления
- **Аргументация:** "Удаление через свайп - интуитивный паттерн для мобильных приложений. Подтверждение удаления предотвращает случайную потерю данных"

**US-044: Удаление устройства из комнаты**
- Показать удаление устройства из комнаты без удаления самого устройства
- **Аргументация:** "Эта функция полезна при перестановках - устройство остается в доме, но может быть перенесено в другую комнату"

#### Итоги работы
- "Я успешно разработал прототип мобильного приложения Stets Home"
- "Прототип реализует все функции MVP, определенные в интегрированном ПМИ"

#### Основные достижения
1. **Полное покрытие MVP:** Все функции из интегрированного ПМИ реализованы в прототипе
2. **Соответствие требованиям:** Прототип соответствует карте пользовательских историй и шагам проверки из ПМИ
3. **UX-решения:** Учтены потребности обеих персон (Михаил и Алёна)
4. **Валидация и обработка ошибок:** Все edge cases обработаны (невалидные данные, дубликаты, лимиты, недоступные устройства)
5. **Дополнительные функции:** Реализованы функции совместного доступа, расширяющие возможности системы
6. **Готовность к разработке:** Прототип может служить техническим заданием для разработчиков

#### Метрики успеха MVP (если уместно)

## Ключевые моменты для аргументации решений

### UX-решения
1. **Два способа добавления устройств:** QR-код (быстро) и ручной ввод с выбором типа (надежность)
2. **Групповое управление:** Удобно для быстрого управления освещением
3. **Валидация в реальном времени:** Улучшает UX, предотвращает ошибки
4. **Свайп для удаления:** Интуитивный паттерн для мобильных приложений
5. **Восстановление пароля через код:** Быстрее, чем через ссылку, код действует 5 минут

### Технические решения
1. **Валидация на клиенте и сервере:** Безопасность и производительность
2. **Обработка офлайн-состояний:** Надежность системы
3. **Лимиты (комнаты, устройства, сценарии):** Предотвращение перегрузки системы
4. **Подтверждение критических действий:** Предотвращение потери данных

### Соответствие персонам
1. **Михаил (Новичок):** Простота, энергосбережение, валидация и подсказки
2. **Алёна (Эксперт):** Сценарии, автоматизация, групповое управление, совместный доступ
