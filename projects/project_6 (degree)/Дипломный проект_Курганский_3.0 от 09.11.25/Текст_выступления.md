## АКТИВНОСТЬ 1: Войти в приложение

### US-001: Регистрация нового пользователя

Регистрация включает валидацию email и пароля на клиенте для улучшения UX. Пароль должен содержать 8-16 символов с обязательными строчными и прописными буквами для безопасности. Проверка совпадения паролей предотвращает ошибки ввода.

### US-003: Вход в приложение

Обработка ошибок входа важна для безопасности. Раздельные сообщения для неверного пароля и несуществующего email помогают пользователю понять проблему.

### US-004: Восстановление пароля

Восстановление пароля через 4-значный код - быстрый и безопасный подход. Код действует 5 минут, что обеспечивает баланс между безопасностью и удобством. Возможность повторной отправки улучшает UX.

### US-005: Изменение email

Изменение email через код подтверждения - безопасный подход. Код отправляется сразу после ввода нового email, что ускоряет процесс по сравнению с письмом.

### US-006: Выход из приложения

При выходе очищаются локальные кеши и токены, сессия закрывается на сервере.

## АКТИВНОСТЬ 2: Организовать умный дом

### US-007: Создание комнат

Комнаты помогают структурировать управление устройствами. Лимит в 10 комнат достаточен для большинства домов.

### US-008: Редактирование комнат

Редактирование комнат позволяет пользователям адаптировать структуру под свои нужды.

### US-009: Удаление комнат

Защита от удаления комнат с устройствами предотвращает потерю данных и улучшает UX.

### US-010: Добавление устройства через QR-код

QR-код - самый быстрый способ добавления устройств. Это критично для достижения метрики успеха: время онбординга < 5 минут. Проверка производителя QR-кода предотвращает добавление несовместимых устройств.

### US-011: Добавление устройства вручную

Ручной ввод необходим, если QR-код поврежден. Выбор типа из справочника обеспечивает корректную категоризацию устройств. Валидация формата предотвращает ошибки.

### US-012: Предотвращение дубликатов устройств

Проверка дубликатов на уровне дома обеспечивает целостность данных.

### US-013: Привязка устройства к комнате

Гибкость привязки устройств позволяет пользователям организовать дом по своему усмотрению.

### US-014: Удаление устройства

Удаление через свайп - интуитивный паттерн для мобильных приложений. Подтверждение удаления предотвращает случайную потерю данных.

### US-044: Удаление устройства из комнаты

Эта функция полезна при перестановках - устройство остается в доме, но может быть перенесено в другую комнату.

## АКТИВНОСТЬ 3: Управлять умным домом

### TR-7: Выдача доступа к дому

Совместный доступ позволяет нескольким пользователям управлять одним домом. Это полезно для семей, где несколько человек могут управлять устройствами.

### TR-7.1: Принятие приглашения

Принятие приглашения открывает доступ к дому. Это позволяет быстро подключать новых пользователей без повторной настройки.

### US-016: Управление режимом энергосбережения

Режим энергосбережения - ключевая функция для персоны Михаила, который заинтересован в экономии энергии. Время отклика увеличивается до 5 секунд, но это приемлемо для энергосберегающего режима.

### US-017: Управление всеми устройствами комнаты

Групповое управление особенно удобно для быстрого управления освещением в комнате. Это улучшает UX для обеих персон.

### US-018: Создание сценария

Сценарии - ключевое конкурентное преимущество и востребованы продвинутыми пользователями (Алёна). Лимит в 10 сценариев достаточен для MVP.

### US-019: Ручной запуск сценария

Ручной запуск позволяет быстро воспроизвести набор действий. Обработка недоступных устройств обеспечивает надежность.

### US-020: Настройка расписания сценария

Сценарии могут создаваться с расписанием или без него. Это дает гибкость: пользователь может сначала создать сценарий, а затем настроить расписание. Расписание - востребованная функция по данным заказчика.

### US-021: Удаление сценария

Удаление через свайп - интуитивный паттерн. Подтверждение удаления предотвращает случайную потерю настроенных автоматизаций.

1. Полное покрытие MVP: Все функции из интегрированного ПМИ реализованы в прототипе.

2. Соответствие требованиям: Прототип соответствует карте пользовательских историй и шагам проверки из ПМИ.

5. Дополнительные функции: Реализованы функции совместного доступа, расширяющие возможности системы.

### Метрики успеха MVP

- Время онбординга: < 5 минут (поддерживается быстрым добавлением через QR)

- Время отклика устройства: < 2 секунды (указано в требованиях)
