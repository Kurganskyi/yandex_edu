# Практические задачи

В этой папке собраны практические задания и упражнения, выполненные в рамках курса "Системный аналитик" от Яндекс Практикум.

---

## 📁 Структура папки

```
tasks/
├── task_1/              # Моделирование процессов для сети клиник «Вита»
├── task_2/              # Расширение API
├── task_3/              # Программа и методика испытаний
├── task_4/              # Постановка задачи для фильтрации и массового закрепления пользователей
├── task_5/              # Фильтрация по эксклюзивности в стриминговом сервисе Otium
├── task_6/              # Telegram Bot для управления таймерами
├── task_7/              # Тестирование веб-приложения «Капсула»
├── task_8/              # ПМИ и Руководство пользователя
└── sql_tasks/           # SQL-задачи
```

---

## 📋 Описание задач

### Task 1: Моделирование процессов для сети клиник «Вита»

**Описание**: Создание диаграмм BPMN (as-is / to-be), выявление возможностей автоматизации, разработка прототипа.

**Нотации и инструменты**: BPMN (as-is/to-be), UML State Machine, UX Design, Figma, прототипирование, анализ процессов

**Структура**:
- `diagrams/` - диаграммы процессов

---

### Task 2: Расширение API

**Описание**: Актуализация C4-диаграммы, проектирование нового API, документация в Swagger YAML.

**Нотации и инструменты**: C4 Model, API Design, Job Stories, REST API, Swagger/OpenAPI, YAML, техническая документация

**Структура**:
- `api/` - спецификации API (Swagger YAML)
- `diagrams/` - C4-диаграммы

---

### Task 3: Программа и методика испытаний

**Описание**: Тестирование, составление ПМИ, руководство пользователя.

**Нотации и инструменты**: Test Plan, Test Cases, ПМИ (Программа и методика испытаний), функциональное тестирование, техническая документация

**Структура**:
- `diagrams/` - диаграммы

---

### Task 4: Постановка задачи для фильтрации и массового закрепления пользователей

**Описание**: Анализ текущих функций, формулировка требований для frontend/backend.

**Нотации и инструменты**: Анализ требований, DevTools, Swagger, Figma, UX проектирование, системный анализ

**Структура**:
- `docs/` - документация
- `diagrams/` - диаграммы
- `templates/` - шаблоны

---

### Task 5: Фильтрация по эксклюзивности в стриминговом сервисе Otium

**Описание**: Анализ архитектуры, изучение SOAP API, доработка XSD схем, диаграммы последовательности.

**Нотации и инструменты**: UML Sequence Diagram, SOAP API, XML/XSD, системный анализ, архитектурное проектирование, интеграции

**Структура**:
- `examples/` - примеры запросов (XML)
- `README.md` - описание задачи

---

### Task 6: Telegram Bot для управления таймерами

**Описание**: Создание полнофункционального бота с inline-кнопками, парсингом времени, обработкой ошибок.

**Нотации и инструменты**: Python, aiogram 3.21.0, asyncio, метапромтинг, разработка ботов, обработка ошибок

**Структура**:
- `bot_v2_meta/` - код бота (финальная версия)
  - `bot.py` - основной файл бота
  - `requirements.txt` - зависимости
- `requirements.txt` - зависимости проекта
- `env_example.txt` - пример конфигурации

**Особенности**:
- Асинхронная архитектура
- Inline-кнопки для управления таймерами
- Парсинг времени в различных форматах
- Обработка ошибок и валидация

---

### Task 7: Тестирование веб-приложения «Капсула»

**Описание**: Функциональное тестирование сервиса бронирования капсульных отелей, составление тест-планов.

**Нотации и инструменты**: Functional Testing, Test Cases, Test Plan, Web Testing, Defect Reports, UX Testing

**Структура**:
- `TEST_PLAN.md` - план тестирования

---

### Task 8: ПМИ и Руководство пользователя

**Описание**: Создание программы и методики испытаний и руководства пользователя по ГОСТ 19 для веб-сайта «Капсула».

**Нотации и инструменты**: ПМИ по ГОСТ 19, техническая документация, руководство пользователя, Test Plan, ГОСТ стандарты

**Структура**:
- `USER_GUIDE.md` - руководство пользователя

---

### SQL Tasks: SQL-задачи

**Описание**: Практические задания по работе с SQL, включая JOIN операции, агрегацию данных и сложные запросы.

**Структура**:
- `8. Operator_INNER_JOIN.md` - задачи по INNER JOIN
- `9. Операторы LEFT OUTER JOIN и RIGHT OUTER JOIN.md` - задачи по LEFT/RIGHT JOIN
- `database_schema.md` - схема базы данных

**Темы**:
- INNER JOIN
- LEFT OUTER JOIN
- RIGHT OUTER JOIN
- Группировка и агрегация
- Подзапросы
- Работа с NULL значениями
