### Назначение папок

- **`projects/`** — полные учебные проекты с детальным описанием, диаграммами и документацией
- **`tasks/`** — небольшие практические задания и упражнения

- **`notes/`** — конспекты, примеры кода и вспомогательные материалы
- **`portfolio/`** — краткие кейсы для быстрого ознакомления работодателей

**Особое внимание:** [`project_6 (degree)/`](./projects/project_6%20(degree)/docs/) — дипломный проект, демонстрирующий полный цикл системного анализа от сбора требований до проектирования архитектуры с использованием современных инструментов и методологий.

## Используемые нотации и инструменты

### Методологии и подходы
- **User-Centered Design (UCD)** - проектирование с фокусом на пользователя
- **Agile/Scrum** - итеративная разработка с разделением на спринты
- **MoSCoW** - приоритизация требований (Must Have, Should Have, Could Have)
- **MVP Framework** - определение минимально жизнеспособного продукта

### Работа с требованиями
- **User Stories** - описание функциональности с точки зрения пользователя
- **User Story Mapping** - структурирование пользовательских историй по эпикам и активностям
- **Gherkin** - формализованные сценарии приемки (Given-When-Then)
- **Job Stories** - описание задач и контекста использования
- **Requirements Matrix** - матрица анализа противоречий требований
- **Personas** - описание целевых пользователей

### Моделирование данных
- **ER-диаграмма** - логическая модель данных в нотации Crow's Foot (Мартина)
- **Нормализация** - приведение БД к 3НФ (третья нормальная форма)
- **Data Dictionary** - словарь данных с типами, ограничениями и валидациями
- **Crow's Foot Notation** - визуализация связей и кардинальности

### Моделирование процессов
- **DFD (Data Flow Diagrams)** - диаграммы потоков данных
- **DFD Level 0** - контекстная диаграмма (система и внешние сущности)
- **DFD Level 1** - логическая диаграмма (детализация процессов)
- **Нотация Гейна-Сарсона** - стандарт для DFD

### Диаграммы и визуализация
- **UML:** Use Case, Sequence Diagram, State Machine, Class Diagrams
- **BPMN** - моделирование бизнес-процессов (as-is/to-be)
- **C4 Model** - архитектурные диаграммы
- **Sitemap** - карта навигации приложения
- **User Flows** - пользовательские сценарии
- **System Architecture** - архитектурные диаграммы

### Техническая документация
- **ПМИ (Программа и методика испытаний)** - по ГОСТ 19
- **ТЗ (Техническое задание)** - спецификация требований
- **ГОСТ 19** - стандарт технической документации для ПО
- **Test Cases** - тест-кейсы и чек-листы
- **Test Plan** - план тестирования
- **Defect Reports** - отчеты о дефектах

### Инструменты визуализации
- **Mermaid** - диаграммы в текстовом формате (DFD, ER, flowchart, sitemap)
- **Figma** - прототипирование и дизайн интерфейса
- **Draw.io** - диаграммы процессов и архитектуры
- **Git** - версионный контроль документации

### API и интеграции
- **REST API** - архитектурный стиль API
- **SOAP API** - протокол обмена данными
- **Swagger/OpenAPI** - спецификация и документация API
- **XML/XSD** - схемы данных для SOAP
- **JSON** - формат обмена данными
- **YAML** - формат конфигурации и спецификаций

### Тестирование
- **Functional Testing** - функциональное тестирование
- **Test-driven Development** - тестирование через Gherkin
- **Regression Testing** - регрессионное тестирование
- **Usability Testing** - тестирование удобства использования
- **Web Testing** - тестирование веб-приложений

### Проектирование интерфейса
- **UX Design** - проектирование пользовательского опыта
- **UI Prototyping** - создание прототипов интерфейса
- **Information Architecture** - структура информации
- **Wireframes** - каркасы интерфейса
- **Mockups** - макеты экранов

### Разработка
- **Python** - разработка Telegram-ботов
- **aiogram 3.21.0** - фреймворк для Telegram ботов
- **asyncio** - асинхронное программирование
- **SQL** - работа с базами данных
- **Метапромтинг** - использование ИИ для генерации технических решений
